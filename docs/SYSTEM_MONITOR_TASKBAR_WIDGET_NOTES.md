# 系统监测任务栏小窗下线记录

## 当前结论

系统监测的 Windows 任务栏小窗能力已在主流程中暂时下线，不再对外提供设置入口，也不再在运行时主动启用。

## 下线原因

此前已经尝试过嵌入式子窗口、overlay、layered window、按像素 alpha 提交等多种实现路径，但在 Windows 任务栏宿主环境下仍持续出现以下问题：

- 任务栏区域间歇性空白或不可见
- 字体发虚、闪动和宿主融合不稳定
- 不同主题、任务栏位置和窗口切换场景下表现不一致

这类问题已经超出“局部修补”范围，继续保留入口只会制造误导和额外维护成本，因此先从运行时和 UI 层下线。

## 保留内容

- helper 端历史实现代码仍保留在 `src/plugins/systemMonitor/windowsHelper/WindowsMonitorHelper/`
- DTO 和设置结构暂未做破坏性删除，方便后续在明确方案后重构接回

## 后续重做建议

如果未来重启该能力，应先完成以下前置工作：

- 重新定义状态 owner 和宿主集成边界
- 基于稳定的任务栏宿主策略先做独立 PoC，再接回主流程
- 先补自动化验证矩阵，再开放用户入口
